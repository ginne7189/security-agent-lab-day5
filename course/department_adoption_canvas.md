# Department Adoption Canvas (course/department_adoption_canvas.md)

Day 5 통합 Security PoC 이후, 우리 팀(부서)에 실제로 적용할 방안을 한 장으로 정리합니다.
각 칸을 1~3줄로 채우세요. 정답이 아니라 "우리 환경에 맞는 결정"을 적는 것이 목표입니다.

---

## 1. 우리 팀 정책은 어디에 들어가는가?
> (예: policies/CLAUDE.md 에 사내 보안 규칙을 주입. 어떤 규칙을 추가/수정할지)

-

## 2. 기존 보안 도구는 어떤 MCP 서버로 감쌀 것인가?
> (예: 사내 SAST/SCA/Secret 스캐너를 security-tools MCP 로 래핑. 어떤 도구부터?)

-

## 3. HITL 기준은 무엇으로 둘 것인가?
> (예: Critical/High 발견 또는 BLOCK 해제 시 관리자 승인 강제. 임계값은?)

-

## 4. Audit Log 는 어디에 저장할 것인가?
> (예: memory/audit_log.jsonl → 사내 SIEM/로그 수집기로 전송. 보존 기간은?)

-

## 5. Final Report 는 누가 검토/승인할 것인가?
> (예: 보안 담당자 검토 → 팀장 승인. 승인 흐름과 책임자)

-

## 6. 먼저 적용할 작은 업무는 무엇인가?
> (예: 특정 레포 1개의 PR 사전 점검부터. 가장 작고 안전한 시작점)

-

## 7. 확장 시 가장 큰 리스크는 무엇인가?
> (예: 오탐으로 인한 개발 흐름 방해, 비밀값 유출, 외부 API 의존. 완화책은?)

-

---

> 연결: Day 1~4 에서 정책·도구·HITL·Audit·위험도 점수화를 각각 체험했습니다.
> 이 캔버스는 그 조각들을 "우리 부서 도입 계획"으로 합치는 마무리 산출물입니다.
